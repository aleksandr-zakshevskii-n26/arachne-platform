## Opportunity: Place Value Exploder

**Target user:** Children aged 6–10 learning multi-digit numbers, and parents/teachers trying to explain place value during homework.

**Problem:** Place value is the single most foundational concept in elementary arithmetic — without it, multi-digit addition, subtraction, multiplication, and division are opaque. Teachers consistently report it as one of the hardest concepts to convey. When a child sees "347," they don't intuitively grasp that the 3 means three hundreds, the 4 means four tens, and the 7 means seven ones. Every existing tool (Math Learning Center, ABCya, IXL) requires the child to *build* the representation or *answer questions about it* — none simply show the answer. A confused 7-year-old searching with a parent finds nothing that just *explains* visually.

**Our angle:** Type a number, instantly see it exploded into sized blocks — large squares for hundreds, medium for tens, small for ones — with the expanded form (300 + 40 + 7 = 347) displayed as text. Zero interaction beyond typing. The visual size encoding creates an immediate "aha" that uniform-sized manipulatives cannot: a hundred *looks* big, a one *looks* small. This reversal — from "prove you understand" to "let me show you" — is a genuine gap in every competitor's approach.

**MVP scope:** Single `index.html` file, no backend, no libraries. One `<input>` field (number, clamped 1–9999). Core logic: parse integer into digits via `toString().split('')`, render colored `<div>` blocks into labeled rows (Thousands / Hundreds / Tens / Ones), display expanded form as text below. Pure DOM manipulation with inline CSS for block sizing. **Estimated build time: 4–6 minutes.** Edge cases are trivial: skip rows where the digit is 0, clamp input at boundaries.

**Why now:** Place value is taught across grades 1–4 in every curriculum worldwide (Common Core, UK National Curriculum, Singapore Math). The concept recurs as numbers scale (ones → thousands), giving the tool multi-year relevance per child. It is also the strongest natural anchor for a future Visual Math Toolkit bundle — if we build one tool first, this is the one that addresses the most fundamental concept and pulls the bundling timeline forward.

**Risks:** (1) The tool may feel too simple — a parent might think "I could just draw this myself." Mitigation: speed and polish matter; instant feedback beats a hand-drawn diagram. (2) Limited replay value — once a child gets it, they stop using it. Acceptable: this is a reference tool, not a game. (3) Visual scaling breaks above 9999 (too many blocks). Mitigation: hard clamp at 4 digits for MVP.

**Recommended next step:** Product director to issue a go/no-go decision. If go, assign to a developer to build the single HTML file and ship within one session.
