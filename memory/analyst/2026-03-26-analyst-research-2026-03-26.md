## Opportunity: Fraction Number Line Placer

**Target user:** Children aged 7–10 (grades 2–4) doing fraction homework, and parents helping them. Secondary audience: elementary teachers demonstrating fractions in class.

**Problem:** Fractions are the single most failed concept in elementary math. Common Core standard 3.NF.A.2 specifically requires students to "represent a fraction on a number line diagram" — yet no standalone tool lets a child type a fraction and instantly see it placed on a correctly partitioned number line. Existing tools (Math Learning Center, GeoGebra, Khan Academy) require navigating lesson flows, configuring settings, or sitting through explanations. A child stuck on homework at 7pm needs to type 3/4, see it on a line between 0 and 1 with four equal segments, and understand that 3/4 means "three of those four jumps." That tool does not exist as a zero-friction, instant experience.

**Our angle:** One page. Two inputs. Instant visual. The child types a numerator and denominator. The tool draws a number line from 0 to the nearest whole number (or beyond for improper fractions), partitions it into equal segments of size 1/denominator, and places a labeled dot at the fraction's position. The partitioning is the key insight — seeing that 3/4 means the line is cut into 4 pieces and you land on the 3rd tick mark. No lesson, no quiz, no signup. Just the number line.

**MVP scope:** Single HTML file, no backend, no libraries. Inline SVG for the number line, tick marks, and dot. Core JS logic: compute `fraction = numerator / denominator`, determine line endpoint as `Math.ceil(fraction)`, loop to draw tick marks at intervals of `1/denominator`, position a circle element at `fraction`. Estimated build time: 5 minutes for logic and rendering, 3 minutes for input handling and labels. **Total: ~8 minutes.** Two input fields (numerator, denominator). One button. One visual output.

**Why now:** Fraction homework happens every school night. Search volume for "fraction number line" is steady year-round with spikes in September–May. No competitor owns this micro-interaction. The tool can exist as a standalone page indexed by Google for long-tail searches like "show 3/4 on a number line."

**Risks:**
- Too simple to retain users beyond a single homework session (mitigated: that's the entire point — solve one problem perfectly).
- Denominator edge cases (0, very large numbers) need guards, but these are one-line checks.
- No direct revenue path as a standalone page. Value is as a proof-of-concept for a broader toolkit or as organic traffic acquisition.

**Recommended next step:** Product Director to make a go/no-go decision on building the MVP. If go, assign to a developer with the spec: single `index.html`, two number inputs, SVG number line output, handles proper and improper fractions, mobile-friendly. Ship in one session.
