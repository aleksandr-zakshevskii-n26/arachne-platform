## Opportunity: Binary Light Bulbs

**Target user:** Children aged 10–14 encountering binary numbers for the first time, whether in school computing classes (UK KS2/KS3, US CSTA standards) or through self-directed curiosity about how computers work.

**Problem:** Binary number representation is abstract and confusing when taught on paper. Children see "10110" and cannot connect it to the number 22. Existing tools are either text-only converters (type decimal, get binary string — no understanding gained) or physical card-based activities (CS Unplugged) that require a teacher and props. There is no clean, single-page, interactive tool where a child can *click light bulbs on and off* and watch the decimal number change in real time — and vice versa.

**Our angle:** A row of 8 light bulbs (circles), each labeled with its place value (128, 64, 32, 16, 8, 4, 2, 1). Click a bulb to toggle it on/off. The decimal total updates instantly. Alternatively, type a number (0–255) and the bulbs flip to show its binary representation. The bidirectional interaction — bulbs↔number — is the "aha" moment. A child *sees* that 22 = 16 + 4 + 2, because exactly those three bulbs light up. The addition breakdown is shown beneath: `16 + 4 + 2 = 22`.

**MVP scope:** Single HTML file, no backend, no libraries. Core logic: an 8-element array of bits, a render function that draws 8 SVG/CSS circles (yellow = on, grey = off), a sum function (`bit[i] * 2^(7-i)`), and a bidirectional sync between the bulb clicks and a number input field. Estimated build time: **7 minutes** for core logic, plus styling. The place-value labels and addition breakdown are static text derived from the same bit array.

**Why now:** Computing education is mandatory or expanding in the UK, Australia, and increasingly the US. Teachers need zero-setup digital tools that work on any device. The gap is real: I found no polished, standalone, click-the-bulbs binary tool on the open web. CS Unplugged remains physical-only. This is an underserved niche with growing demand.

**Risks:**
- The concept skews older (10+), missing the 6–9 age band entirely.
- Engagement ceiling is moderate: once the child "gets it," they may stop after 2–3 minutes. There's no creative sandbox element.
- Monetization as a standalone tool is weak. Value accrues only if it anchors a collection of CS-concept tools.

**Recommended next step:** Product director decides: (1) build this as a standalone proof-of-concept to validate the format, or (2) scope it as the first in a "CS Unplugged — Digital Edition" series, which changes the positioning and naming.
