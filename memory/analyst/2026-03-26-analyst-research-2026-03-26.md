## Opportunity: Binary Lightbulb Visualiser

**Target user:** Children aged 10–14 learning computing fundamentals, particularly UK KS3 students (Year 7–9) and US middle schoolers encountering binary for the first time. Secondary audience: curious younger kids (8+) who want to understand "how computers think."

**Problem:** Binary number representation is one of the first abstract concepts in computing education, and it's where many kids hit a wall. Existing tools are text-in/text-out converters — a child types 42, sees "101010," and has no intuition for *why*. Teachers resort to physical cards (CS Unplugged) or hand-drawn diagrams, but there is no clean, interactive web tool that bridges the gap between typing a number and *seeing* its binary structure instantly.

**Our angle:** A single-page tool where 8 lightbulbs represent 8 bits. A child types a number (0–255) and the bulbs instantly light up or stay dark. Each bulb is labeled with its place value (128, 64, 32, 16, 8, 4, 2, 1). Below the bulbs, a sum equation appears: e.g., "42 = 32 + 8 + 2." Crucially, it works bidirectionally — kids can also *click* bulbs to toggle them on/off and watch the decimal number update live. This click-to-explore mode is where the "aha" happens: kids discover patterns (every number that ends in a lit "1" bulb is odd, doubling a number shifts all bulbs left). No existing tool does this with a child-friendly lightbulb metaphor.

**MVP scope:** Single `index.html` file, no backend, no libraries. Core logic is `number.toString(2).padStart(8, '0')` plus a loop toggling CSS classes on 8 div elements. Lightbulb glow is CSS `border-radius` + `box-shadow`. Estimated core build time: **3–4 minutes**. Full polish (labels, sum equation, click-to-toggle, responsive layout): under 10 minutes.

**Why now:** Computing education is expanding globally. The UK mandates binary in KS3; the US CSTA standards include it for grades 6–8. The CAS (Computing at School) community has 40,000+ teacher members actively seeking interactive resources. Binary is taught annually to every new cohort — demand is structural, not seasonal. The CS Unplugged lightbulb/card metaphor is proven over 20+ years but has never been executed well as a focused web micro-tool.

**Risks:** (1) The concept is simple enough that a teacher could build it themselves — our moat is execution quality, not idea novelty. (2) Binary is one unit within computing; standalone demand may be narrow. (3) Free alternatives could emerge quickly once the format is validated.

**Recommended next step:** Product director to make a go/no-go decision on building the MVP. If go: assign to a developer with a brief specifying single HTML file, 8-lightbulb layout, bidirectional interaction (type number ↔ click bulbs), labeled place values, and sum equation display.
